# Bedolaga Continuity

Модуль удержания пользователей для **Bedolaga** и **Remnawave**. После окончания оплаченной
подписки он сохраняет заранее настроенный ограниченный доступ к Telegram, Mini App и оплате, а
после продления автоматически возвращает обычный профиль.

Проект помогает не терять связь с клиентом в момент, когда ему особенно нужен доступ к боту и
странице оплаты. Все изменения вводятся поэтапно: сначала проверка, затем резервная копия,
наблюдение, тест на одном пользователе и только после подтверждения — контролируемое включение.

## Возможности

- интерактивное меню и пошаговый мастер первоначальной настройки;
- обнаружение Bedolaga и локальной панели Remnawave с обязательным подтверждением адреса;
- выбор существующего internal squad или безопасное создание нового из выбранных inbound;
- ограничение срока и объёма трафика Continuity-доступа;
- автоматическое возвращение полного доступа после оплаты;
- установка в выключенном режиме без немедленного влияния на пользователей;
- проверка на одном выбранном пользователе и rollout ступенями 5/25/50/100%;
- резервная копия, диагностический архив и безопасный откат;
- обработка больших баз пакетами без создания тестовых пользователей в рабочей базе.

> Текущий пакет совместимости предназначен для точного upstream-коммита **Bedolaga v3.64.0**.
> Неизвестная версия или изменённые затрагиваемые файлы блокируют установку до любых изменений.

## Как это работает

```text
Подписка активна  → обычный профиль
Подписка истекла  → ограниченный Continuity-профиль
Оплата прошла     → обычный профиль восстановлен
```

Безопасный путь запуска:

```text
preflight → backup → install (disabled) → observe → canary → 5% → 25% → 50% → 100%
```

Установка не включает режим сразу для всего пула. Каждый опасный переход требует отдельного
действия и явного подтверждения администратора.

## Что подготовить

- Linux-сервер с Docker и Docker Compose v2;
- установленную Bedolaga v3.64.0;
- URL и API-ключ Remnawave;
- отдельный inbound ограниченного доступа на ноде;
- маршрутизацию к Telegram, Mini App, DNS и оплате на стороне этого inbound/профиля.

Мастер может создать или выбрать **internal squad**, но не переписывает существующие Xray-профили,
хосты и ноды. Это намеренное ограничение: сетевую политику нужно проверить до выдачи доступа
пользователям.

## Установка

Скачайте установщик и его контрольную сумму из последнего релиза:

```bash
curl -fsSLO https://github.com/LaRsonOFFai/bedolaga-grace-bridge/releases/latest/download/install.sh
curl -fsSLO https://github.com/LaRsonOFFai/bedolaga-grace-bridge/releases/latest/download/SHA256SUMS
grep ' install.sh$' SHA256SUMS | sha256sum -c -
less install.sh
sudo bash install.sh
```

Установщик запускает мастер конфигурации, выполняет проверку совместимости без изменений и затем
открывает меню. В дальнейшем меню доступно командой:

```bash
sudo gracectl
```

Для автоматизации доступны отдельные команды:

```bash
sudo gracectl configure
sudo gracectl preflight
sudo gracectl install
sudo gracectl observe
sudo gracectl canary
sudo gracectl approve-canary
sudo gracectl activate
sudo gracectl pause
sudo gracectl resume
sudo gracectl status
sudo gracectl bundle
sudo gracectl rollback
sudo gracebridge-rescue
```

Подробности: [установка](docs/installation.md), [архитектура](docs/architecture.md),
[безопасный откат](docs/recovery.md), [совместимость](docs/compatibility.md) и
[проверка на 40 000 записей](docs/load-testing.md).
