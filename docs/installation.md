# Установка

## Требования

- Ubuntu/Debian Linux;
- установка выполняется на сервере, где запущен Telegram-бот Bedolaga;
- Docker и Docker Compose v2;
- точный upstream commit Bedolaga v3.64.0 из матрицы совместимости;
- PostgreSQL;
- действующий Remnawave API key;
- отдельный inbound ограниченного доступа; internal squad можно выбрать или создать в мастере;
- минимум 2 GiB свободного места для дампа и образов.

## Этапы

1. Скачать `install.sh` и `SHA256SUMS` из GitHub Release.
2. Проверить SHA-256 и просмотреть скрипт.
3. На сервере с ботом Bedolaga запустить `sudo bash install.sh`.
4. Автоматический мастер найдёт Bedolaga и возможный адрес панели Remnawave.
5. Мастер проверит API и предложит выбрать или создать internal squad.
6. Указать срок и лимит ограниченного доступа; секреты сохранятся с правами `0600`.
7. Мастер выполнит read-only проверку совместимости.
8. После подтверждения он создаст дамп и установит выключенный кандидат.
9. Мастер включит наблюдение без записи и запросит UUID одного тестового пользователя.
10. После проверки теста мастер предложит ступени 5%, 25%, 50% и 100%.
11. На каждой ступени можно продолжить, остановиться, поставить активации на паузу или откатиться.

Панель Remnawave может находиться на другом сервере. На сервер панели и VPN-ноды устанавливать
Continuity не требуется. Если мастер был закрыт, продолжите командой `sudo gracectl wizard`.

Ни `install.sh`, ни `gracectl install` не активируют Continuity для клиентов без прохождения
canary и явного подтверждения администратора.

## Пользовательская правка auth

Отдельная правка, позволяющая кабинету принимать старый токен, сохранится, если
её файл не входит в патч Grace. Если код Bedolaga изменён в одном из
затрагиваемых файлов, установка остановится, а не перезапишет правку.
