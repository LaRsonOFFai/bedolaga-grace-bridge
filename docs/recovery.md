# Откат и аварийное восстановление

Обычный путь:

```bash
sudo gracectl rollback
```

Если основная CLI недоступна:

```bash
sudo gracebridge-rescue
```

Обычный `gracectl rollback` выполняется в следующем порядке:

1. остановка новых активаций;
2. отмена pending/processing команд;
3. проверка текущего panel tag;
4. восстановление снимка только при tag `GRACE_ACCESS`;
5. пропуск пользователей, у которых уже появилась оплаченная подписка;
6. возврат исходного Compose и образа Bedolaga;
7. проверка запуска;
8. запись результата в локальное состояние.

Автономный `gracebridge-rescue` используется, если сломана основная CLI. Он
проверяет контрольные суммы последнего бэкапа, возвращает исходные файлы и
запускает Bedolaga только по базовому Compose. Он намеренно не меняет
пользовательские overlay без работающей логики сверки; после восстановления CLI
нужно выполнить обычный `gracectl rollback`.

`database.dump` не восстанавливается автоматически: полный `pg_restore` может
стереть оплаты, прошедшие после создания копии. Он используется только для
ручного аварийного восстановления после отдельного анализа.
